## ❌ 严重问题（需立即修复）

### 问题1: 缺乏单元测试 ❌ **高优先级**

**现状**:
```
项目中没有 tests/ 目录
没有任何单元测试
```

**风险**:
- 重构时无法验证正确性
- Bug 容易反复出现
- 新人不敢改代码

**建议**: 见 `testing_strategy.md`

## ⚠️ 中等问题（计划改进）

### 问题4: 数据流不清晰 ⚠️

**现状**: 数据在函数间传递复杂

```python
# 数据经过多层转换
raw_data → preprocessed_data → train_df → scaled_data → model_input
# 中间步骤难以追踪
```

**建议**:
- 引入数据类（dataclass）封装
- 使用 Pipeline 模式

---

### 问题5: 异常处理不一致 ⚠️

**问题代码**:
```python
# 风格1: 静默捕获
try:
    result = some_function()
except Exception:
    pass  # ← 错误被吞掉

# 风格2: 打印日志
except Exception as e:
    print(f"Error: {e}")  # ← 没有记录到日志文件

# 风格3: 抛出错误
except Exception as e:
    st.error(f"失败: {str(e)}")  # ← 只在 UI 显示
```

**建议**:
- 统一使用 logging 模块
- 建立错误分级机制
- 关键错误需要持久化记录
