# 项目架构图和数据流程图

## 当前架构摘要

当前代码已完成应用层拆分：

```text
app.py                         # Streamlit 入口和 Tab 路由
app/pages/                     # 页面层：训练、预测、微调、上传模型预测
app/services/                  # 服务层：训练服务、预测服务
app/components/                # UI 展示组件
app/utils/                     # Session state 等工具
app/ui_helpers.py              # 跨页面共享辅助函数
ml_trader/                     # 核心算法包
```

下方历史架构图仍描述核心数据流和模块职责，其中应用层已由单一 `app.py` 拆分为 `app.py + app/pages + app/services`。

## 4. 模型微调策略说明

### 4.1 冻结策略（MLP）
- **不冻结**：所有层参数可训练
- **冻结第一层**：fc1层固定
- **冻结第二层**：fc2层固定
- **冻结所有层**：仅微调不生效
- **部分冻结**：fc1前半部分冻结

### 4.2 冻结策略（Transformer）
- **不冻结**：全模型可训练
- **冻结输入层**：input_linear固定
- **冻结编码器层**：除最后一层外的encoder固定
- **冻结输出层**：fc层固定
- **冻结所有层**：仅微调不生效

### 4.3 混合训练
- 新数据：微调区间数据
- 旧数据：原训练集按比例采样
- 混合比例：0.1~2.0倍可调

## 5. 性能优化点

1. **缓存机制**：预测结果缓存，策略参数调整无需重新预测
2. **并行训练**：多组模型可并行训练（受限于硬件）
3. **早停机制**：验证集监控，避免过拟合
4. **模型持久化**：支持下载/上传 .pkl 模型文件
5. **增量预测**：只对新数据区间进行预测
